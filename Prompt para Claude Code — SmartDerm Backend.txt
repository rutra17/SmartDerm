# Prompt para Claude Code — SmartDerm Backend

Cole este prompt inteiro no Claude Code dentro da pasta `SmartDerm-Julia/BackEnd`.

---

## Contexto do projeto

Estou desenvolvendo o **SmartDerm**, uma plataforma de análise dermatológica com IA. O backend é Node.js com Express (ESModules, `"type": "module"`). Já existem os seguintes arquivos funcionando:

- `src/server.js` — servidor Express com CORS e rota `/api`
- `src/routes/imageRoutes.js` — rota `POST /api/upload`
- `src/controllers/imageController.js` — orquestra o fluxo: busca prompt no Supabase → chama IA → salva resposta
- `src/services/geminiService.js` — integração com Gemini 2.5 Flash (recebe buffer, mimeType, prompt → retorna texto)
- `src/services/supabase.js` — cliente Supabase inicializado com `SUPABASE_URL` e `SUPABASE_KEY` do `.env`
- `src/middlewares/upload.js` — Multer com `memoryStorage`, campo `imagem`

## O que preciso que você implemente

### 1. `src/models/` — Esquemas de dados (objetos JS puros, sem ORM)

Crie um arquivo por entidade. Cada modelo é uma função que retorna um objeto validado/formatado, pronto para inserir no Supabase.

**`src/models/usuarioModel.js`**
- Campos: `id`, `nome`, `email`, `tipo_conta` (enum: `'paciente'`, `'medico'`, `'cientista'`), `identificador` (CPF, CRM ou username), `created_at`
- Exporte uma função `criarUsuario(dados)` que valida os campos obrigatórios e retorna o objeto formatado
- Exporte uma função `formatarUsuario(raw)` que limpa o objeto vindo do banco (remove campos sensíveis)

**`src/models/laudoModel.js`**
- Campos: `id`, `consulta_id`, `paciente_id`, `medico_id`, `texto_ia`, `ia_utilizada`, `prompt_utilizado`, `imagem_url`, `created_at`
- Exporte `criarLaudo(dados)` com validação dos campos obrigatórios
- Exporte `formatarLaudo(raw)` que retorna um objeto limpo

**`src/models/consultaModel.js`**
- Campos: `id`, `paciente_id`, `medico_id`, `status` (enum: `'aberta'`, `'em_analise'`, `'concluida'`), `titulo`, `created_at`, `updated_at`
- Exporte `criarConsulta(dados)` e `formatarConsulta(raw)`

**`src/models/mensagemModel.js`**
- Campos: `id`, `consulta_id`, `role` (enum: `'user'`, `'assistant'`), `texto`, `ia_utilizada`, `prompt_utilizado`, `imagem_url`, `created_at`
- Exporte `criarMensagem(dados)` e `formatarMensagem(raw)`

### 2. `src/providers/ai/` — Provedores de IA (interface unificada)

Cada arquivo deve exportar **uma única função** com a mesma assinatura:
```js
analisarImagem(imageBuffer, mimeType, promptFinal) → Promise<string>
```

**`src/providers/ai/gemini.js`**
- Use `@google/generative-ai` com o modelo `gemini-2.5-flash`
- Converta o buffer para base64 com `inlineData`
- Baseie-se no `geminiService.js` já existente (que pode ser aposentado depois)

**`src/providers/ai/openai.js`**
- Use o SDK `openai` com o modelo `gpt-4o`
- Envie a imagem como `image_url` com `data:${mimeType};base64,...`
- Mensagens no formato `[{ role: 'user', content: [{ type: 'image_url', ... }, { type: 'text', ... }] }]`

**`src/providers/ai/claude.js`**
- Use `@anthropic-ai/sdk` com o modelo `claude-opus-4-5`
- Envie a imagem como `{ type: 'image', source: { type: 'base64', media_type: mimeType, data: base64 } }`
- Retorne `response.content[0].text`

**`src/providers/ai/deepseek.js`**
- DeepSeek usa a mesma interface da OpenAI (`baseURL: 'https://api.deepseek.com'`)
- Use o modelo `deepseek-chat`
- Envie a imagem como base64 da mesma forma que o OpenAI
- Use a chave `DEEPSEEK_API_KEY`

**`src/providers/ai/index.js`** — Factory/roteador
```js
// Deve exportar uma função:
// getProviderIA(nomeModelo) → retorna a função analisarImagem do provider correto
// nomeModelo aceita: 'gemini', 'openai', 'claude', 'deepseek'
// Se não reconhecer, usa 'gemini' como fallback com um console.warn
```

### 3. `src/providers/storage/` — Storage de imagens com MinIO

**`src/providers/storage/minio.js`**
- Inicialize o cliente MinIO com as variáveis de ambiente: `MINIO_ENDPOINT`, `MINIO_PORT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`
- Use `useSSL: false` por padrão (sobrescrevível por env)
- Exporte as seguintes funções:

```js
// Faz upload de um buffer e retorna a URL pública do arquivo
uploadImagem(buffer, mimeType, nomeArquivo) → Promise<string>

// Gera um nome de arquivo único: `laudos/{timestamp}-{uuid-curto}.{extensao}`
gerarNomeArquivo(mimeType) → string

// Verifica se o bucket existe e cria se não existir
garantirBucket(nomeBucket) → Promise<void>
```

- No upload, chame `garantirBucket` antes de fazer o put
- A URL pública deve ter o formato: `http://${MINIO_ENDPOINT}:${MINIO_PORT}/${MINIO_BUCKET}/${nomeArquivo}`

**`src/providers/storage/index.js`**
```js
// Exporta diretamente as funções do minio.js
// Facilita trocar de provider no futuro sem mudar os imports nos controllers
export { uploadImagem, gerarNomeArquivo } from './minio.js';
```

### 4. Atualize o `src/controllers/imageController.js`

Depois de criar os providers, atualize o controller para:

1. Importar `getProviderIA` de `../providers/ai/index.js` em vez de chamar `geminiService` diretamente
2. Importar `uploadImagem`, `gerarNomeArquivo` de `../providers/storage/index.js`
3. Importar `criarMensagem` de `../models/mensagemModel.js`
4. No fluxo:
   - Antes de chamar a IA, faça upload da imagem para o MinIO e capture a `imagemUrl`
   - Use `getProviderIA(aiModel)` para pegar a função de análise correta baseada no `aiModel` que vem do body
   - Ao salvar no Supabase, inclua o campo `imagem_url: imagemUrl` na inserção
   - Use `criarMensagem({...})` para montar o objeto antes de inserir

Mantenha todos os `console.log` existentes e adicione logs para o upload do MinIO.

## Regras de implementação

- Todos os arquivos devem usar **ESModules** (`import`/`export`, sem `require`)
- Todo código deve ter **try/catch** com erros re-lançados ou logados
- Não use TypeScript, não instale novas dependências além das já listadas
- Não modifique `server.js`, `imageRoutes.js`, `upload.js`, `supabase.js`
- O `geminiService.js` existente pode continuar existindo (não delete), mas o controller deve migrar para usar o provider novo
- Comente em português, código em inglês (nomes de variáveis/funções)
